# Project Log - 2026-05-09 - Phase 3 JSON Dual-Field Support

## Task Overview
Implemented Phase 3 JSON Dual-Field Support for the "grove" to "project" transition. This involved updating several core structs to support both legacy and new JSON keys for backward compatibility.

## Changes

### 1. Custom Marshaling/Unmarshaling Pattern
Implemented the pattern described in `.scratch/sub-design-wire-protocol.md`:
- New Go fields already exist from Phase 1.
- Updated JSON tags to use new `project` terminology.
- Added `MarshalJSON` to emit both new and legacy keys.
- Added `UnmarshalJSON` to accept both new and legacy keys, with precedence for the new keys.

### 2. Files Updated

## Verification Results
- `go build` for affected packages succeeded.
- Verified custom marshaling logic follows the shadow field pattern to avoid recursion.

## Observations
- Many structs shared similar patterns, making the implementation repetitive but straightforward.
- `ResolvedSecret` was present in both `pkg/api` and `pkg/hubclient`, requiring identical updates.
- The use of `type Alias T` within `MarshalJSON`/`UnmarshalJSON` is effective for overriding specific fields without manual mapping of all fields.
